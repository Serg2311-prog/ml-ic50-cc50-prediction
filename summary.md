# Итоговый анализ проекта: IC50, CC50 и SI

## 1. Цель и постановка задачи

Проект посвящен сравнению классических моделей машинного обучения для задач
drug discovery на молекулярных дескрипторах. Рассматриваются две группы задач:

- **регрессия**: прогноз численных значений `IC50, mM`, `CC50, mM`, `SI`;
- **классификация**:
  - `IC50 > median`;
  - `CC50 > median`;
  - `SI > median`;
  - `SI > 8`.

Финальная версия проекта использует строгий **descriptor-only** подход:
экспериментальные endpoints (`IC50`, `CC50`, `SI`) и любые target-like
производные удаляются из матрицы признаков `X` для всех задач.

---

## 2. EDA: основные выводы

EDA показал, что данные пригодны для классического ML, но требуют аккуратной
интерпретации:

1. **Целевые переменные асимметричны.**
   - `IC50` и `CC50` имеют правостороннюю асимметрию.
   - `SI` наиболее тяжелохвостый, потому что является отношением `CC50 / IC50`.

2. **Выбросы биологически значимы.**
   Экстремальные значения могут соответствовать наиболее активным или токсичным
   соединениям, поэтому их нельзя механически удалять без предметной проверки.

3. **Мультиколлинеарность ожидаема.**
   Молекулярные дескрипторы часто описывают близкие физико-химические свойства,
   поэтому часть признаков коррелирует. Для деревьев это менее критично, а для
   линейных моделей может ухудшать устойчивость коэффициентов.

4. **`SI` является производным endpoint.**
   Так как `SI = CC50 / IC50`, использование `IC50`, `CC50`, `SI` или их
   логарифмов как признаков приводит к data leakage.

5. **Descriptor-only постановка научно корректнее.**
   В production-like сценарии значения endpoints заранее неизвестны. Модель
   должна использовать только молекулярные дескрипторы, доступные до эксперимента.

---

## 3. Исправление data leakage

### Где была проблема

В notebook workflow после EDA создавалась колонка:

```python
df["IC50_log"] = np.log10(df["IC50, mM"] + 1e-8)
```

Затем она попадала в признаки. Для задачи `IC50 > median` это означало, что
модель получала почти прямое преобразование таргета.

### Почему старые ROC-AUC ~0.99-1.00 ошибочны

`IC50_log` монотонно связан с `IC50`, поэтому бинарный target
`IC50 > median` почти полностью восстанавливается порогом по `IC50_log`.
Деревья находили этот порог напрямую, а Logistic Regression получала почти
линейно отделимый признак.

Такие метрики не отражали способность модели прогнозировать активность по
молекулярным дескрипторам. Это был leakage-артефакт.

### Как исправлено

- введен единый список target-like колонок;
- добавлен pattern-based detector для `ic50`/`cc50` и безопасный список
  SI-производных;
- `build_feature_matrix()` удаляет все target-like признаки;
- перед обучением выполняется runtime-проверка `validate_no_target_like_columns`;
- notebook outputs со старыми leakage-метриками очищены;
- все финальные таблицы используют только post-leakage результаты.

## 4. Регрессия

### 4.1 IC50

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Random Forest | **457.77** | **240.54** | **0.3718** |
| Gradient Boosting | 463.36 | 254.44 | 0.3563 |
| Linear Regression | 508.25 | 267.18 | 0.2256 |

**Вывод.** Лучшей моделью по RMSE стал Random Forest. Зависимость `IC50`
от дескрипторов нелинейна, поэтому ансамбли деревьев превосходят линейную модель.

### 4.2 CC50

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Random Forest | **455.70** | 287.39 | **0.5995** |
| Gradient Boosting | 455.71 | **283.71** | 0.5994 |
| Linear Regression | 566.87 | 390.94 | 0.3802 |

**Вывод.** `CC50` прогнозируется лучше остальных regression targets. Random
Forest является лучшей моделью по RMSE, а Gradient Boosting практически равен
ему по RMSE и немного лучше по MAE.

### 4.3 SI

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Gradient Boosting | **1353.61** | 198.02 | **0.0878** |
| Random Forest | 1358.42 | **191.19** | 0.0813 |
| Linear Regression | 1379.37 | 222.30 | 0.0528 |

**Вывод.** Регрессия `SI` остается сложной: тяжелый хвост и отношение двух
экспериментальных величин делают таргет шумным и плохо аппроксимируемым только
по дескрипторам.

## 5. Классификация после исправления leakage

Все classification-результаты ниже являются post-leakage результатами. Старые
значения ROC-AUC около `0.99-1.00` исключены из финальных выводов. Полный набор
Accuracy/Precision/Recall/F1/PR-AUC сохранен в детальных
`results/classification_*/model_comparison.csv`; в этой сводке для компактности
показаны ROC-AUC и лучшая модель.

| Задача | Logistic Regression | Random Forest | Gradient Boosting | Лучшая модель |
|---|---:|---:|---:|---|
| `IC50 > median` | 0.75 | **0.78** | 0.77 | Random Forest |
| `CC50 > median` | 0.8297 | **0.8310** | 0.8149 | Random Forest |
| `SI > median` | 0.65 | **0.69** | 0.67 | Random Forest |
| `SI > 8` | 0.67 | **0.75** | 0.74 | Random Forest |

### Интерпретация

- `IC50 > median`: Random Forest лидирует, но Gradient Boosting близок. Это
  указывает на нелинейную, но не тривиальную связь дескрипторов с активностью.
- `CC50 > median`: Logistic Regression практически не уступает Random Forest
  (ROC-AUC 0.8297 vs 0.8310), значит в задаче есть выраженный линейный
  компонент. Random Forest остается лучшей моделью и предпочтительным
  production-like baseline за счет большей устойчивости к нелинейностям.
- `SI > median`: качество умеренное; SI сложнее, потому что является
  производным и тяжелохвостым endpoint.
- `SI > 8`: Random Forest снова лучший. Пороговая задача по высокому SI
  практически важна, поэтому интерпретация должна учитывать дисбаланс классов
  и PR-AUC из детального CSV.

## 6. Сравнение моделей

### Почему ансамбли деревьев чаще лучше Logistic Regression

- зависимости между дескрипторами и endpoints нелинейны;
- деревья естественно моделируют пороги и локальные области признакового
  пространства;
- Random Forest устойчив к шуму, выбросам и коррелированным признакам;
- Gradient Boosting может точнее аппроксимировать сложные зависимости, но
  чувствительнее к настройке гиперпараметров;
- Logistic Regression полезна как baseline, но ограничена линейной границей
  решения.

### Реально лучшая модель

По совокупности classification-задач лучшим production-like baseline является
**Random Forest**. Он лидирует по ROC-AUC во всех четырех classification
задачах и при этом лучше соответствует ожидаемой нелинейной
структуре связи между дескрипторами и endpoints.

---

## 7. Практическая ценность

Проект показывает, что классические ML-модели могут давать полезный baseline
для предварительного скрининга соединений, но качество зависит от постановки:

- `CC50` прогнозируется надежнее;
- `IC50` имеет умеренно предсказуемый сигнал;
- `SI` наиболее сложен из-за отношения `CC50 / IC50` и тяжелого хвоста;
- leakage-контроль критичен: без него метрики становятся недостоверными.
