# Итоговый анализ моделей (IC50, CC50, SI)

## 1. Кратко о данных и постановке

- Объект: химические соединения, описанные числовыми молекулярными дескрипторами.
- Размер данных после удаления служебного индекса: **1001 x 213**.
- Цели:
  - регрессия: `IC50, mM`, `CC50, mM`, `SI`;
  - классификация: `IC50 > median`, `CC50 > median`, `SI > median`, `SI > 8`.

Критическое ограничение anti-leakage:
- в задачах `SI` не используются признаки `IC50` и `CC50`;
- в задачах `IC50/CC50` не используется признак `SI`.

---

## 2. Что показал EDA

1. Данные в основном чистые: всего **36 пропусков** в ограниченном наборе колонок.  
   Выбран подход: `SimpleImputer(strategy="median")`.

2. Целевые переменные асимметричны:
   - `IC50`: skew ≈ **3.67**
   - `CC50`: skew ≈ **1.97**
   - `SI`: skew ≈ **18.01** (самая тяжелохвостая цель)

3. По IQR-критерию доля выбросов:
   - `IC50`: **14.7%**
   - `CC50`: **3.9%**
   - `SI`: **12.5%**

4. Выбросы не удалялись жестко, потому что в задаче drug discovery экстремальные значения могут быть биологически важными (редкие, но перспективные соединения).

## 3. Регрессия: сравнение моделей

Использовались:
- Linear Regression
- Random Forest Regressor
- Gradient Boosting Regressor

Для всех моделей: кросс-валидация (5-fold) + подбор гиперпараметров (`GridSearchCV` / `RandomizedSearchCV`).

### 3.1 IC50 (регрессия)

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Gradient Boosting | **415.49** | 208.81 | **0.4825** |
| Random Forest | 418.91 | **206.03** | 0.4739 |
| Linear Regression | 441.22 | 245.14 | 0.4164 |

**Вывод:** лучшая по RMSE — **Gradient Boosting**. Линейная модель уступает, что ожидаемо при нелинейных зависимостях и асимметричном таргете.

### 3.2 CC50 (регрессия)

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Random Forest | **415.75** | **235.86** | **0.6666** |
| Gradient Boosting | 427.01 | 246.49 | 0.6483 |
| Linear Regression | 446.26 | 325.56 | 0.6159 |

**Вывод:** для `CC50` лучше всего сработал **Random Forest**. По `R2` задача заметно проще, чем `IC50`.

### 3.3 SI (регрессия)

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Gradient Boosting | **1353.47** | 198.30 | **0.0880** |
| Random Forest | 1358.54 | **191.30** | 0.0812 |
| Linear Regression | 1379.37 | 222.30 | 0.0528 |

**Вывод:** регрессия `SI` получилась слабой у всех моделей (`R2` близок к 0.1). Причина — крайне тяжелохвостое распределение `SI` и, вероятно, недостаток информативных признаков именно для этого таргета.

## 5. Какие подходы сработали лучше и почему

1. **Ансамбли деревьев (Random Forest / Gradient Boosting)** чаще выигрывают у линейных моделей:
   - лучше улавливают нелинейные взаимодействия дескрипторов;
   - устойчивее к негауссовым распределениям и выбросам.

2. Для `CC50` качество выше, чем для `IC50` и особенно `SI`:
   - вероятно, связь `CC50` с доступными дескрипторами более стабильна;
   - для `SI` сигнал сложнее из-за сложной формулы и хвостов распределения.

3. Логистическая регрессия дает полезный baseline, но реже становится лидером по ROC-AUC.

## 6. Рекомендации по улучшению проекта

1. **Feature engineering:**
   - добавить агрегаты и отношения дескрипторов;
   - попробовать селекцию признаков (mutual information, permutation importance, SHAP-based filtering).

2. **Работа с SI:**
   - протестировать лог-преобразование (`log1p`) таргета в регрессии;
   - отдельно анализировать верхний хвост (`SI > 8`, `SI > 10`) и рассмотреть ранжирование.

3. **Алгоритмы:**
   - расширить сравнение за счет XGBoost/LightGBM/CatBoost;
   - сделать калибровку вероятностей для классификации (особенно в SI-задачах).

4. **Валидация:**
   - добавить repeated CV и доверительные интервалы метрик;
   - проверить переносимость на внешнем/временном hold-out наборе.

5. **Данные:**
   - увеличить размер выборки, особенно для редких высокоэффективных соединений;
   - проверить качество исходных измерений и consistency протоколов.
