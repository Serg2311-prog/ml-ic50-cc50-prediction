## 1. Постановка задачи

Цель работы - сравнить классические модели машинного обучения для прогнозирования
биологических endpoints по молекулярным дескрипторам.

Рассматривались семь задач:

1. регрессия `IC50, mM`;
2. регрессия `CC50, mM`;
3. регрессия `SI`;
4. классификация `IC50 > median`;
5. классификация `CC50 > median`;
6. классификация `SI > median`;
7. классификация `SI > 8`.

Для всех задач итоговая постановка является **descriptor-only**: модель получает
только молекулярные дескрипторы, доступные до эксперимента. Endpoint-колонки и
их производные исключаются из `X`.

- для моделей применяется `SimpleImputer(strategy="median")`;

Целевые переменные имеют выраженную правостороннюю асимметрию:

| `IC50, mM` | ~3.67 | правый хвост, часть соединений имеет крупные значения |
| `CC50, mM` | ~1.97 | асимметрия умереннее, чем у IC50 и SI |
| `SI` | ~18.01 | самый тяжелый хвост, так как `SI = CC50 / IC50` |

В `eda.ipynb` добавлены графики:

`log1p` заметно сжимает правые хвосты и делает распределения ближе к

### 2.5 Корреляции, мультиколлинеарность и связь признаков с target

Молекулярные дескрипторы часто коррелируют друг с другом, поскольку описывают
близкие физико-химические свойства. Это ожидаемая мультиколлинеарность:

- линейные модели чувствительны к коррелированным признакам и масштабу;
- Random Forest и Gradient Boosting устойчивее, но importance может
  распределяться между похожими признаками;
- для будущей интерпретации полезны permutation importance и SHAP.

В `eda.ipynb` добавлен correlation heatmap для target-переменных и наиболее
связанных descriptor-only признаков. Невысокие линейные корреляции объясняют,
почему деревья и бустинг в среднем предпочтительнее линейных моделей.

## 3. Anti-leakage analysis

### 3.1 Почему endpoints нельзя использовать как признаки

`SI` является функцией `IC50` и `CC50`:

```text
SI = CC50 / IC50
```

Следовательно:

- при прогнозировании `SI` нельзя использовать `IC50` и `CC50`;
- при прогнозировании `IC50` или `CC50` нельзя использовать `SI`;
- в production-like сценарии безопаснее удалить **все** endpoints из `X` для
  любой задачи.

Причина: значения `IC50`, `CC50`, `SI` неизвестны до эксперимента. Если они
попадают в признаки, модель получает информацию, которая в реальном применении
недоступна.

### 3.2 Найденная утечка

В notebook workflow была создана колонка:

```python
df["IC50_log"] = np.log10(df["IC50, mM"] + 1e-8)
```

Затем эта колонка попадала в `X`. Для задачи `IC50 > median` это почти прямое
раскрытие ответа: логарифм является монотонным преобразованием `IC50`, поэтому
порог по `IC50_log` практически восстанавливает бинарный target.

### 3.3 Почему ROC-AUC около 1.0 был некорректен

Старые значения ROC-AUC около `0.99-1.00` возникали не потому, что модели
идеально решали химическую задачу, а потому что в признаки попадала производная
target-переменной. Деревья могли найти порог по `IC50_log`, а Logistic Regression
получала почти линейно отделимый признак.

Такая оценка не переносится на новые молекулы, потому что в реальном сценарии
`IC50_log` заранее неизвестен.

### 3.4 Как leakage устранен

В проекте реализована системная защита:

- в `ml_common.py` задан список target-like колонок;
- `get_target_like_columns()` удаляет явные targets и производные по имени;
- для `ic50` и `cc50` используется поиск по подстроке;
- для `SI` используется безопасный список точных target-like имен
  (`SI`, `SI_log`, `log_SI`, `SI_bin`, `SI_class`, `SI_scaled`, `target_SI`);
- `build_feature_matrix()` строит descriptor-only `X`;
- `validate_no_target_like_columns()` выбрасывает `ValueError`, если leakage
  обнаружен перед обучением;
- notebook outputs со старыми leakage-результатами очищены.

## 6. Подробные результаты regression

### 6.1 IC50 regression

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Gradient Boosting | **415.49** | 208.81 | **0.4825** |
| Random Forest | 418.91 | **206.03** | 0.4739 |
| Linear Regression | 441.22 | 245.14 | 0.4164 |

Лучшей моделью по RMSE стал Gradient Boosting. Random Forest близок по качеству
и имеет лучший MAE. Это указывает на нелинейную связь активности с
дескрипторами и на ограниченность линейной модели.

### 6.2 CC50 regression

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Random Forest | **415.75** | **235.86** | **0.6666** |
| Gradient Boosting | 427.01 | 246.49 | 0.6483 |
| Linear Regression | 446.26 | 325.56 | 0.6159 |

`CC50` прогнозируется наиболее стабильно: R2 выше, чем для `IC50` и `SI`.
Random Forest показывает лучший баланс ошибки и устойчивости.

### 6.3 SI regression

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|
| Gradient Boosting | **1353.47** | 198.30 | **0.0880** |
| Random Forest | 1358.54 | **191.30** | 0.0812 |
| Linear Regression | 1379.37 | 222.30 | 0.0528 |

Регрессия `SI` остается слабой. Это ожидаемо: `SI` имеет экстремальный хвост и
является отношением двух экспериментальных величин. Для этой задачи более
перспективны robust losses, log-transform target или пороговая classification.

## 7. Подробные результаты classification

### 7.1 IC50 > median

| Модель | ROC-AUC |
|---|---:|
| Random Forest | **0.78** |
| Gradient Boosting | 0.77 |
| Logistic Regression | 0.75 |

После исправления leakage качество стало реалистичным. Random Forest лидирует,
но Gradient Boosting близок; это говорит о наличии нелинейного, но не
тривиального сигнала в дескрипторах.

### 7.2 CC50 > median

| Модель | ROC-AUC |
|---|---:|
| Random Forest | **0.83** |
| Logistic Regression | **0.83** |
| Gradient Boosting | 0.81 |

Это самая стабильная classification задача. Logistic Regression достигает
качества Random Forest, что указывает на выраженный линейный компонент. Random
Forest остается предпочтительным baseline, потому что лучше защищен от
нелинейностей и взаимодействий.

### 7.3 SI > median

| Модель | ROC-AUC |
|---|---:|
| Random Forest | **0.69** |
| Gradient Boosting | 0.67 |
| Logistic Regression | 0.65 |

Качество умеренное. `SI` является производным endpoint и имеет тяжелый хвост,
поэтому сигнал по descriptor-only признакам слабее.

### 7.4 SI > 8

| Модель | ROC-AUC |
|---|---:|
| Random Forest | **0.75** |
| Gradient Boosting | 0.74 |
| Logistic Regression | 0.67 |

Порог `SI > 8` практически важен для поиска высокоселективных соединений.
Random Forest лучше ранжирует объекты по вероятности высокого `SI`, а Gradient
Boosting является близкой альтернативой.

## 9. Переобучение и устойчивость

Прямые train/test gaps для финальной версии не сохранены в `results/*.csv`, но
по структуре задач можно выделить риски:

- старые train/test ROC-AUC около `0.99-1.00` были не переобучением, а leakage;
- после descriptor-only исправления ROC-AUC находится в диапазоне `0.65-0.83`,
  что выглядит реалистично;
- деревья и бустинг могут переобучаться при слишком большой глубине, поэтому в
  коде используется cross-validation / randomized search;
- `SI` наиболее нестабилен из-за тяжелого хвоста.

Для более строгой оценки переобучения следует сохранять train и test metrics
для всех моделей в `results/`.

## 10. Важные признаки

Финальный notebook строит feature importance для Random Forest и Gradient
Boosting. Численный список top features не сохранен отдельным CSV, поэтому в
отчете не фиксируются конкретные ранги признаков без выполнения notebook.

Методологический вывод:

- важными должны быть только descriptor-only признаки;
- endpoint-колонки (`IC50`, `CC50`, `SI`) и производные (`IC50_log`, `SI_log`,
  и т.д.) не должны попадать в importance;
- если target-like колонка появляется в top features, это является индикатором
  новой утечки.
