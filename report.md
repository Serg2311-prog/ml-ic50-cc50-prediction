## 1. Постановка задачи

Цель работы - сравнить классические модели машинного обучения для прогнозирования
биологических конечных показателей по молекулярным дескрипторам.

Рассматривались семь задач:

1. регрессия `IC50, mM`;
2. регрессия `CC50, mM`;
3. регрессия `SI`;
4. классификация `IC50 > median`;
5. классификация `CC50 > median`;
6. классификация `SI > median`;
7. классификация `SI > 8`.

Для всех задач итоговая постановка использует только молекулярные дескрипторы:
модель получает признаки, доступные до эксперимента. Колонки с конечными
показателями и их производные исключаются из матрицы признаков.

## 2. Описание данных

### 2.1 Размер выборки и признаки

Датасет загружается из стабильного пути:

```text

```

После удаления технического индекса `Unnamed: 0` данные содержат:

- **1001 объект**;
- **213 колонок**;
- 3 колонки конечных показателей: `IC50, mM`, `CC50, mM`, `SI`;
- около 210 числовых молекулярных дескрипторов.

Загрузка централизована в `ml_common.py`, поэтому все эксперименты используют
один и тот же источник данных и единые правила предотвращения утечки целевой
переменной.

### 2.2 Пропуски

EDA показывает, что пропусков немного: **36 пропущенных значений** в ограниченном
наборе колонок. Массовой деградации качества данных нет. Поэтому выбран
консервативный подход:

- строки не удаляются;
- для моделей применяется `SimpleImputer(strategy="median")`;
- медианная импутация предпочтительна из-за асимметрии распределений и выбросов.

### 2.3 Распределения целевых переменных

Целевые переменные имеют выраженную правостороннюю асимметрию:

| Target | Skewness | Интерпретация |
|---|---:|---|
| `IC50, mM` | ~3.67 | правый хвост, часть соединений имеет крупные значения |
| `CC50, mM` | ~1.97 | асимметрия умереннее, чем у IC50 и SI |
| `SI` | ~18.01 | самый тяжелый хвост, так как `SI = CC50 / IC50` |

В `eda.ipynb` представлены графики:

- гистограммы с KDE для `IC50`, `CC50`, `SI`;
- сравнение распределений **до** и **после** `log1p`;
- таблица skewness до/после `log1p`.

`log1p` заметно сжимает правые хвосты и делает распределения ближе к
симметричным. Логарифмы целевых переменных используются только для
исследовательского анализа или как возможная трансформация целевой переменной;
они не используются как признаки.

### 2.4 Выбросы

По IQR-критерию доля выбросов для target-переменных:

| Target | Доля IQR-выбросов |
|---|---:|
| `IC50, mM` | ~14.7% |
| `CC50, mM` | ~3.9% |
| `SI` | ~12.5% |

В `eda.ipynb` представлены диаграммы размаха для:

- `IC50`, `CC50`, `SI`;
- `MolWt`, `TPSA`, `MolLogP`, `qed`, `NumHDonors`, `NumHAcceptors`.

Агрессивное удаление выбросов не применялось. В drug discovery экстремальные
значения могут соответствовать наиболее интересным соединениям: высокоактивным,
высокотоксичным или высокоселективным. Удалять такие точки без предметной
проверки методологически опасно.

### 2.5 Корреляции, мультиколлинеарность и связь признаков с целевыми переменными

Молекулярные дескрипторы часто коррелируют друг с другом, поскольку описывают
близкие физико-химические свойства. Это ожидаемая мультиколлинеарность:

- линейные модели чувствительны к коррелированным признакам и масштабу;
- Random Forest и Gradient Boosting устойчивее, но importance может
  распределяться между похожими признаками;
- для будущей интерпретации полезны permutation importance и SHAP.

В `eda.ipynb` представлена тепловая карта корреляций для целевых переменных и
наиболее связанных дескрипторов. Невысокие линейные корреляции объясняют,
почему деревья и бустинг в среднем предпочтительнее линейных моделей.

## 3. Анализ утечки целевой переменной

### 3.1 Почему конечные показатели нельзя использовать как признаки

`SI` является функцией `IC50` и `CC50`:

```text
SI = CC50 / IC50
```

Следовательно:

- при прогнозировании `SI` нельзя использовать `IC50` и `CC50`;
- при прогнозировании `IC50` или `CC50` нельзя использовать `SI`;
- в сценарии практического применения безопаснее удалить **все** конечные
  показатели из `X` для любой задачи.

Причина: значения `IC50`, `CC50`, `SI` неизвестны до эксперимента. Если они
попадают в признаки, модель получает информацию, которая в реальном применении
недоступна.

### 3.2 Найденная утечка

В исходном исследовательском ноутбуке была создана колонка:

```python
df["IC50_log"] = np.log10(df["IC50, mM"] + 1e-8)
```

Затем эта колонка попадала в `X`. Для задачи `IC50 > median` это почти прямое
раскрытие ответа: логарифм является монотонным преобразованием `IC50`, поэтому
порог по `IC50_log` практически восстанавливает бинарную целевую переменную.

### 3.3 Почему ROC-AUC около 1.0 был некорректен

Старые значения ROC-AUC около `0.99-1.00` возникали не потому, что модели
идеально решали химическую задачу, а потому что в признаки попадала производная
целевой переменной. Деревья могли найти порог по `IC50_log`, а Logistic Regression
получала почти линейно отделимый признак.

Такая оценка не переносится на новые молекулы, потому что в реальном сценарии
`IC50_log` заранее неизвестен.

### 3.4 Как устранена утечка

В проекте реализована системная защита:

- в `ml_common.py` задан список колонок, связанных с целевыми переменными;
- `get_target_like_columns()` удаляет явные целевые переменные и производные по имени;
- для `ic50` и `cc50` используется поиск по подстроке;
- для `SI` используется безопасный список точных имен производных
  (`SI`, `SI_log`, `log_SI`, `SI_bin`, `SI_class`, `SI_scaled`, `target_SI`);
- `build_feature_matrix()` строит матрицу признаков только по молекулярным
  дескрипторам;
- `validate_no_target_like_columns()` останавливает обучение, если перед
  построением модели обнаруживаются признаки утечки;
- устаревшие результаты с утечкой исключены из финальной интерпретации.

## 6. Подробные результаты регрессии

### 6.1 Регрессия IC50

| Модель | RMSE | MAE | R2 |
|---|---:|---:|---:|

Лучшей моделью по RMSE стал Random Forest. Gradient Boosting близок по качеству,

| Linear Regression | 566.87 | 390.94 | 0.3802 |

`CC50` прогнозируется наиболее стабильно: R2 выше, чем для `IC50` и `SI`.
Random Forest является лучшей моделью по RMSE, а Gradient Boosting практически
равен ему по RMSE и немного лучше по MAE.

### 6.3 Регрессия SI

является отношением двух экспериментальных величин. Для этой задачи более
перспективны устойчивые функции потерь, логарифмическое преобразование целевой
переменной или переход к пороговой классификации.

## 9. Переобучение и устойчивость

Разрывы между обучающей и тестовой выборками не являются главным предметом
данного отчета, однако по структуре задач можно выделить риски:

- старые ROC-AUC около `0.99-1.00` были не переобучением, а следствием утечки
  целевой переменной;
- после исключения конечных показателей из признаков ROC-AUC находится в
  диапазоне `0.6543-0.8310`, что является более реалистичной оценкой;
- деревья и бустинг могут переобучаться при слишком большой глубине, поэтому в
  коде используется перекрестная проверка и случайный поиск гиперпараметров;
- `SI` наиболее нестабилен из-за тяжелого хвоста.

Для более строгой оценки переобучения в дальнейшем следует отдельно сохранять
метрики на обучающей и тестовой выборках для всех моделей.

## 10. Важные признаки

Финальный ноутбук строит диаграммы важности признаков для Random Forest и
Gradient Boosting. В отчете не фиксируются конкретные ранги признаков, поскольку

Методологический вывод:

- важными должны быть только молекулярные дескрипторы;
- колонки конечных показателей (`IC50`, `CC50`, `SI`) и производные
  (`IC50_log`, `SI_log` и т.д.) не должны попадать в список важных признаков;
- появление такой колонки среди важных признаков является индикатором новой

## 12. Что можно улучшить дальше

1. Сохранять разрывы между обучающей и тестовой выборками для всех моделей.
2. Добавить повторную перекрестную проверку и доверительные интервалы.

4. Добавить калибровочные кривые для вероятностей классификации.
