# Курсовой проект: классическое ML для IC50, CC50 и SI

Проект посвящен сравнению классических методов машинного обучения для задач drug discovery:
- регрессия `IC50`, `CC50`, `SI`;
- бинарная классификация (`IC50 > median`, `CC50 > median`, `SI > median`, `SI > 8`).

## Цели проекта

1. Провести осмысленный EDA химических дескрипторов и целевых переменных.
2. Построить воспроизводимый пайплайн предобработки.
3. Сравнить несколько моделей для каждой постановки задачи.
4. Дать интерпретируемые выводы и практические рекомендации.

## Структура репозитория

- `data/Данные_для_курсовой_Классическое_МО.xlsx` — исходный датасет для запуска экспериментов. Если файл не включен в репозиторий, его нужно поместить в эту папку вручную после `git clone`.
- `eda.ipynb` — отдельный ноутбук с исследовательским анализом.
- `ml_common.py` — общие утилиты (загрузка данных, anti-leakage, split, метрики).
- `regression_utils.py` — общий код регрессионных экспериментов.
- `classification_utils.py` — общий код классификационных экспериментов.
- `ic50_regression.py`, `cc50_regression.py`, `si_regression.py` — запуск регрессии.
- `classification_ic50.py`, `classification_cc50.py`, `classification_si.py` — запуск классификации.
- `results/` — таблицы метрик, предсказания, матрицы ошибок и выводы по каждой задаче.
- `summary.md` — итоговый сравнительный анализ по всем моделям.
- `report.md` — аналитический отчет с разбором leakage, ограничений и рекомендаций.

## Методология

### 1) EDA
- проверка структуры данных (shape, типы);
- анализ пропусков;
- описательная статистика;
- гистограммы распределений;
- boxplot для выбросов;
- корреляционный анализ (heatmap);
- разбор таргетов `IC50`, `CC50`, `SI`.

### 2) Предобработка
- удаление технической колонки `Unnamed: 0`;
- обработка пропусков: `SimpleImputer(strategy="median")`;
- масштабирование (`StandardScaler`) для линейных моделей;
- train/test split: `test_size=0.2`, `random_state=42`;
- защита от утечки целевой переменной:
  - при построении `X` используется descriptor-only подход;
  - из признаков удаляются `IC50`, `CC50`, `SI` и их производные target-like колонки.

### 3) Модели

**Регрессия (для каждого таргета):**
- Linear Regression
- Random Forest Regressor
- Gradient Boosting Regressor

**Классификация (для каждой задачи):**
- Logistic Regression
- Random Forest Classifier
- Gradient Boosting Classifier

Подбор гиперпараметров: `GridSearchCV` и `RandomizedSearchCV` с 5-fold CV.

## Финальные результаты

Проект использует descriptor-only постановку: при построении `X` удаляются
`IC50`, `CC50`, `SI` и все target-like производные признаки. Это исключает
утечку целевой переменной и делает оценку ближе к production-like сценарию,
где экспериментальные endpoints неизвестны заранее.

### Регрессия

| Задача | Лучшая модель | RMSE | MAE | R2 |
|---|---|---:|---:|---:|
| IC50 | Gradient Boosting | 415.49 | 208.81 | 0.4825 |
| CC50 | Random Forest | 415.75 | 235.86 | 0.6666 |
| SI | Gradient Boosting | 1353.47 | 198.30 | 0.0880 |

### Классификация после исправления leakage

| Задача | Лучшая модель | ROC-AUC |
|---|---|---:|
| IC50 > median | Random Forest | ~0.78 |
| CC50 > median | Random Forest / Logistic Regression | ~0.83 |
| SI > median | Random Forest | ~0.69 |
| SI > 8 | Random Forest | ~0.75 |

Старые notebook outputs с ROC-AUC около `0.99-1.0` были leakage-артефактом:
в признаки попадал `IC50_log`, то есть прямое преобразование target. Эти
значения удалены из актуальных результатов и не используются в выводах.

Детальная методология, причины leakage, интерпретация результатов и дальнейшие
рекомендации описаны в `summary.md` и `report.md`.

## Воспроизводимость

Рекомендуемые зависимости:
- `pandas`, `scikit-learn`, `numpy`, `matplotlib`, `seaborn`, `openpyxl`, `nbformat`

### Подготовка после `git clone`

1. Установите зависимости:

```bash
python3 -m pip install -r requirements.txt
```

2. Убедитесь, что датасет лежит по стабильному пути:

```text
data/Данные_для_курсовой_Классическое_МО.xlsx
```

Все скрипты загружают данные через `ml_common.py`, поэтому запуск из корня проекта воспроизводимо использует именно этот файл.

### Запуск экспериментов

Регрессионные задачи:

```bash
python3 ic50_regression.py
python3 cc50_regression.py
python3 si_regression.py
```

Классификационные задачи:

```bash
python3 classification_ic50.py
python3 classification_cc50.py
python3 classification_si.py
```
